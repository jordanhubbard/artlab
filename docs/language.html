Language Reference

Artlab DSL v1.0 — formal grammar (EBNF) and code-generation notes

Program Structure

An Artlab source file is a sequence of statements. There is no required entry point; the runtime looks for exported functions named setup and update by convention. 

# EBNF: program ::= { stmt } EOF

use "artlab/geometry"
use "artlab/lights"

fn setup() {
  # called once when the scene loads
}

fn update(dt: num) {
  # called every frame; dt is the frame delta in seconds
}

Newlines act as statement terminators. Semicolons are optional — you may use them to place multiple statements on one line. Indentation is insignificant (only newlines matter).

Comments

Line comments begin with # and extend to the end of the line. Block comments are not supported in v1.

# this is a comment
let x = 42  # inline comment

Imports — use

# EBNF:
# use_stmt   ::= 'use' use_source [ 'as' IDENT ] newline
# use_source ::= STRING
#              | 'url' ':' STRING
#              | 'embedded' ':' STRING

Three import forms are available:

Stdlib import

use "artlab/math"
use "artlab/geometry" as geo
Compiles to:
import * as _m0 from '/artlab/stdlib/math.js'
import * as geo from '/artlab/stdlib/geometry.js'

Remote URL import

use url:"https://example.com/mylib.js" as myLib
Compiles to:
import * as myLib from "https://example.com/mylib.js"

Embedded (in-package) import

use embedded:"libs/helpers.art" as helpers
Compiles to:
import * as helpers from new URL("libs/helpers.art", import.meta.url)

The optional as IDENT clause binds the module namespace to a local name. Without it, the compiler generates an internal binding (_m0, _m1, ...).

Variables — let

# EBNF: var_decl ::= 'let' IDENT [ ':' TYPE ] '=' expr newline

let radius: num   = 3.14
let name: str     = "hello"
let pos: vec3     = vec3(0, 1, 0)
let active: bool = true
let n = 10        # type annotation is optional
Compiles to:
let radius = 3.14
let pos = new THREE.Vector3(0, 1, 0)

All let bindings are mutable. There is no const equivalent in v1.

Functions — fn

# EBNF:
# fn_def     ::= 'fn' IDENT '(' [ param_list ] ')' [ ':' TYPE ] block
# param_list ::= param { ',' param }
# param      ::= IDENT [ ':' TYPE ]

fn add(a: num, b: num): num {
  return a + b
}

fn greet(name: str) {
  # no return type → void
}
Compiles to:
Top-level functions with no special meaning are compiled as plain JS functions.
Functions named setup / update are compiled as export function setup() {} so the runtime can find them.

Parameters may omit the type annotation. Return type annotation is also optional.

Assignment

# EBNF:
# assign_stmt ::= lvalue '=' expr newline
# lvalue      ::= IDENT { '.' IDENT }

x = 42
planet.position.x = 10
mesh.material.color = color(1, 0, 0)

Assignment targets (l-values) support arbitrary dot-access chains, making it easy to reach into Three.js object hierarchies directly.

Control Flow — if / else

# EBNF: if_stmt ::= 'if' expr block [ 'else' ( if_stmt | block ) ]

if x > 0 {
  doSomething()
} else if x == 0 {
  doOther()
} else {
  doFallback()
}

Braces are required around each branch. There is no ternary operator in v1; use an if/else expression block instead.

Control Flow — loop

# EBNF: loop_stmt ::= 'loop' IDENT 'from' expr 'to' expr [ 'by' expr ] block

# count 0, 1, 2, ... 9
loop i from 0 to 10 {
  doWork(i)
}

# step by 2
loop i from 0 to 10 by 2 {
  doWork(i)
}
Compiles to:
for (let i = 0; i < 10; i++) { ... }
for (let i = 0; i < 10; i += 2) { ... }

The upper bound is exclusive. The loop variable is scoped to the block.

Control Flow — every

# EBNF: every_stmt ::= 'every' expr block
# expr is the interval in seconds

every 0.5 {
  spawnParticle()
}

every 10 {
  logStats()
}
Compiles to:
setInterval(() => { spawnParticle() }, 500)

The interval expression is evaluated once at parse time and converted to milliseconds.

Control Flow — return

# EBNF: return_stmt ::= 'return' [ expr ] newline

fn clamp01(v: num): num {
  if v < 0 { return 0 }
  if v > 1 { return 1 }
  return v
}

A bare return (no expression) returns undefined.

Operators

Operators are listed from lowest to highest precedence:

PrecedenceOperatorsAssociativityNotes
1 (lowest)||leftlogical or
2&&leftlogical and
3== !=leftequality
4< > <= >=leftrelational
5+ -leftadditive
6* / %leftmultiplicative
7- !rightunary negate / not
8**rightexponentiation
9 (highest)() . []leftcall, member, index
 
# examples
let a = 2 ** 10          # 1024  (right-associative)
let b = -a + 1            # -1023
let c = a > 100 && b != 0  # true

Built-in Constructors

# EBNF:
# constructor_expr ::= 'vec2'  '(' expr ',' expr ')'
#                    | 'vec3'  '(' expr ',' expr ',' expr ')'
#                    | 'vec4'  '(' expr ',' expr ',' expr ',' expr ')'
#                    | 'color' '(' expr ',' expr ',' expr ')'
#                    | 'quat'  '(' expr ',' expr ',' expr ',' expr ')'

let uv  = vec2(0.5, 0.5)
let pos = vec3(1, 0, 0)
let red = color(1, 0, 0)
let rot = quat(0, 0, 0, 1)
Compiles to (Three.js):
new THREE.Vector2(0.5, 0.5)
new THREE.Vector3(1, 0, 0)
new THREE.Color(1, 0, 0)
new THREE.Quaternion(0, 0, 0, 1)

Constructor keywords are also valid type annotations (see Built-in Types).

Literals

Number literals

42       # integer
3.14     # float
1.5e-3   # scientific notation

String literals

"hello, world"
"escape sequences: \n \t \\ \" \u0041"

Boolean and null

true   false   null

Array literals

let nums = [1, 2, 3]
let mix  = ["a", 1, true]

Object literals

let opts = { color: 0xff0000, roughness: 0.4, wireframe: false }
let pt   = { x: 1, y: 2, z: 3 }

Type annotations are optional everywhere; the compiler does not enforce them at runtime — they serve as documentation and are reserved for future type-checking tooling.

Reserved Keywords

The following identifiers cannot be used as variable or function names:

use  fn  let  if  else  loop  from  to  by  every  return
true  false  null
num  bool  str  vec2  vec3  vec4  color  quat  mesh  scene
