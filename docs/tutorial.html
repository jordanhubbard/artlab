Tutorial: Your first Artlab package

Build a rotating cube, add lights, hook it up to audio, and export it — in 5 steps.

1
Create a new package

An Artlab package is a directory containing an artlab.json manifest and at least one .js entry file. The simplest way to get started is to open the IDE and create a new package from the file tree.

Start the dev server:

git clone <repo-url> artlab
cd artlab && npm install && npm run dev
# open http://localhost:5173

In the IDE, click + in the file tree to create a new directory under examples/. Add two files:

artlab.json
{
  "name":        "my-scene",
  "version":     "1.0.0",
  "entry":       "my-scene.js",
  "description": "A simple rotating cube"
}
my-scene.js
export function setup(ctx) {}
export function update(ctx, dt) {}

Click the package in the sidebar to load it — you should see a black canvas in the preview pane.

2
Add a rotating cube

The ctx object is injected by the runtime. It gives you Three (the full three.js module), scene, camera, and ctx.add() to add objects and track them for teardown.

my-scene.js
let cube

export function setup(ctx) {
  const { Three } = ctx
  cube = ctx.add(new Three.Mesh(
    new Three.BoxGeometry(2, 2, 2),
    new Three.MeshStandardMaterial({ color: 0x4488ff, roughness: 0.4 })
  ))
  ctx.camera.position.set(0, 2, 6)
}

export function update(ctx, dt) {
  cube.rotation.y += dt * 0.5
}
What you should see: A blue cube slowly rotating. It looks flat because there are no lights yet.

ctx.add(obj) adds the object to the scene and returns it. Using add() instead of scene.add() directly lets the IDE clean up automatically when the package reloads.

3
Add lights

Import the stdlib geometry and lights helpers for convenience, or use Three.js constructors directly. Both work identically.

my-scene.js (updated)
import { sphere, mesh } from '../../src/stdlib/geometry.js'
import { directional, ambient } from '../../src/stdlib/lights.js'

let ball

export function setup(ctx) {
  ball = ctx.add(mesh(sphere(1.5, 64), { color: 0x22aaff, roughness: 0.2, metalness: 0.7 }))

  const sun = ctx.add(directional(0xffffff, 2))
  sun.position.set(10, 20, 10)
  ctx.add(ambient(0x334466, 0.6))

  ctx.camera.position.set(0, 2, 6)
}

export function update(ctx, dt) {
  ball.rotation.y += dt * 0.5
}
What you should see: A properly lit metallic sphere. Drag to orbit, scroll to zoom — orbit controls are wired up by default.
4
Add audio reactivity

Import the audio stdlib module to access FFT band values. Audio requires a user gesture before the context can start — attach it to a click.

Note: Browsers block audio autoplay. Always call audio.start() from a user-gesture handler (click, keydown).
my-scene.js (updated)
import { sphere, mesh } from '../../src/stdlib/geometry.js'
import { directional, ambient } from '../../src/stdlib/lights.js'
import * as audio from '../../src/stdlib/audio.js'

let ball

export function setup(ctx) {
  ball = ctx.add(mesh(sphere(1.5, 64), { color: 0x22aaff, roughness: 0.2, metalness: 0.7 }))

  const sun = ctx.add(directional(0xffffff, 2))
  sun.position.set(10, 20, 10)
  ctx.add(ambient(0x334466, 0.6))

  ctx.camera.position.set(0, 2, 6)

  // unlock audio on first click
  ctx.renderer.domElement.addEventListener('click', () => audio.start(), { once: true })
}

export function update(ctx, dt) {
  audio.update()
  const bass = audio.band('bass')   // 0..1
  const high = audio.band('high')

  // pulse size with bass, spin faster with treble
  const s = 1 + bass * 1.5
  ball.scale.setScalar(s)
  ball.rotation.y += dt * (0.3 + high * 2)
}

export function teardown(ctx) {
  audio.stop()
}
What you should see: A metallic sphere that pulses in size with bass frequencies and spins faster during high-frequency moments. Click the canvas to unlock the audio context first.
5
Export and share

Click the Export button in the IDE toolbar (the download icon) to download your package as a .zip. The archive contains the manifest, all source files, and any assets — everything needed to run the package.

The .zip can be dragged back into the IDE to reload it, shared with teammates, or run standalone via StandaloneRunner.
